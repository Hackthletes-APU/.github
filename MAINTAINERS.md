## Responsibilities

Maintainers are expected to:

- Review pull requests within a reasonable timeframe
- Triage and label new issues
- Help beginners get started
- Enforce the [Code of Conduct](CODE_OF_CONDUCT.md)
- Escalate security reports via [SECURITY.md](SECURITY.md)

## How to become a maintainer

1. Contribute consistently to one or more org repositories
2. Demonstrate good collaboration and code review habits
3. Be nominated by an existing maintainer or project lead
4. Be added to `@Hackthletes-APU/maintainers` by an organisation owner
